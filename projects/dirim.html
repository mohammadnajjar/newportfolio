Dirim

Methods & journey

Dirim is an e-commerce marketplace with a twist — it's designed for fast, local delivery in the UAE. Three separate mobile apps needed to work in perfect sync: the customer browsing and ordering, the seller managing their inventory and listings, and the delivery driver picking up and dropping off. My role was to architect and build the entire backend that ties them together.

The challenge

Three different user types, three different apps, one backend. Every order goes through multiple state transitions — placed, confirmed, picked, in-transit, delivered, returned — and each transition needs to instantly notify all three parties in real time. Meanwhile, the web platform at dirim-mall.com needed the same backend to serve a full web shopping experience.

  • Single API serving 3 Flutter apps and 1 web platform simultaneously
  • Real-time order status pushed to customer, seller, and driver instantly
  • Multi-vendor seller onboarding, product catalog, and fulfilment management
  • Delivery assignment engine: auto-assign driver by proximity and capacity
  • Integrated payment gateway with split settlement to sellers
  • Admin dashboard for orders, disputes, payouts, and analytics

The architecture

Laravel as the API core with MySQL for the relational layer — products, orders, users, wallets. Laravel Echo + Pusher handle all real-time events, pushing order status updates to the correct parties the moment state changes. The driver assignment algorithm queries by geo-distance using Haversine formula and assigns based on driver load and proximity.

For the web platform, Blade templates with server-side rendering kept the shopping experience fast and SEO-friendly, while pulling from the same product and order APIs as the mobile apps.

What shipped

Three production Flutter apps (Dirim, Dirim Store, Dirim Delivery), the dirim-mall.com web platform, a full admin dashboard, and a seller portal. Payment processing, driver tracking, and real-time notifications all live and serving real orders.

The outcome

Sellers onboarded in minutes through the self-service portal. Customers get live delivery tracking from the moment an order is confirmed. The ops team monitors all active deliveries from a single dispatch dashboard with no manual coordination required.

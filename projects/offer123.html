Offer 123

Methods & journey

Offer 123 is an offers and deals discovery app — merchants publish time-limited promotions, users browse and redeem them through the app. The product lives entirely in its backend: the quality of the offer targeting, the redemption validation, and the merchant tools determine whether the platform creates value for both sides.

The challenge

A deals platform needs to balance two very different user types through a single API: merchants managing their promotions and users browsing and redeeming them. The core technical challenge is redemption integrity — an offer must be redeemable exactly the allowed number of times, under the right conditions, with no double-spends.

  • Merchant onboarding: business profile, category selection, branch locations
  • Offer creation: title, description, terms, discount value, valid period, redemption limit
  • Offer scheduling: active / inactive / expired states with automatic transitions
  • User targeting: by location, category preference, and usage history
  • Geo-based discovery: nearby offers sorted by distance using location data
  • Redemption flow: QR code generation, merchant-side scanning, per-user limit enforcement
  • Push notifications: new offers matching user preferences, expiry reminders

The architecture

Laravel with MySQL. Offer state managed as an enum with automatic transitions triggered by scheduled jobs — no manual intervention needed to activate or expire offers at the right time. Redemption uses a database-level unique constraint per (user, offer) pair to prevent race conditions on simultaneous redemption attempts.

Geo-based discovery uses MySQL's spatial index on branch coordinates, returning offers sorted by distance within a configurable radius. Merchant dashboard built with Filament — merchants log in and manage everything themselves without needing support tickets.

What shipped

A production API serving iOS and Android: merchant self-service portal, offer lifecycle management, geo-based offer discovery, QR code redemption with race-condition-safe validation, and push notifications — delivered in 3 months from kick-off to production.

The outcome

Merchants publish and manage their own offers without developer involvement. Users discover nearby deals in real time. Redemptions are validated instantly with zero double-spend incidents since launch.

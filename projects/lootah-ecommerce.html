Lootah E-com

Methods & journey

Lootah E-commerce is the primary retail mobile app for YS Lootah Group, one of Dubai's diversified conglomerates. The app needed a backend that could handle the scale of an enterprise retail operation: wide product catalog, complex pricing rules, multiple delivery zones, and a clean API for the Flutter team to build against.

The challenge

Enterprise retail backends have layers that simple shop backends don't: category hierarchies, SKU-level inventory, tiered pricing, promotional rules, multi-address delivery management, and reporting for operations teams — all while maintaining a fast, consistent API for mobile clients.

  • Deep product catalog: categories, subcategories, variants, SKU management
  • Cart engine: multi-item, quantity management, price recalculation on each update
  • Promo codes and discount rules with stacking restrictions
  • Multi-address management with delivery zone validation
  • Payment integration: card, wallet, and cash on delivery flows
  • Order lifecycle with real-time status updates pushed to the app
  • Reporting endpoints for the operations team

The architecture

Laravel with a clean repository pattern to keep business logic out of controllers. Product data modeled with a flexible attribute system to handle varying product types without schema migrations for each new category. Cart state is persisted server-side tied to the user session — no trust issues with client-side totals hitting the payment gateway.

Pricing rules evaluated through a pipeline: base price → category discount → promo code → loyalty tier. Every applied rule is logged for the receipt and for fraud review. Order status transitions are event-driven with Firebase Cloud Messaging firing on each state change.

What shipped

A production API powering the iOS and Android apps: full product catalog with variant management, server-side cart, multi-payment checkout, promotional rules engine, delivery tracking, and push notification alerts for every order milestone.

The outcome

The Flutter team received a stable, well-documented API with consistent response shapes across every endpoint. Orders process correctly on first attempt. The operations team monitors order flow in real time without needing developer involvement for day-to-day issues.

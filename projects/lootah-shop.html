Lootah Shop

Methods & journey

Most projects I work on have a split: I build the backend, a Flutter developer builds the app. Lootah Shop was different. The client trusted me with the entire product — backend, mobile app, and admin panel. That meant more responsibility, more decisions, and a result I can point to with full ownership.

The challenge

Building a complete retail platform solo means every layer has to be designed to scale — a rushed decision on the data model or API contract becomes technical debt across three separate surfaces at once.

  • Product catalog: categories, subcategories, variants, images, pricing tiers
  • Cart & checkout: guest and authenticated flows, promo codes, delivery address management
  • Order lifecycle: placed → confirmed → packed → dispatched → delivered → returned
  • Payment integration: card checkout, cash on delivery, refund handling
  • Flutter mobile app: all screens, state management, API integration, push notifications
  • Admin dashboard: inventory, orders, customer management, reports, promotions

The architecture

Laravel for the backend with a resource-based REST API designed for the Flutter client first — typed responses, consistent error envelopes, and versioned endpoints. The Flutter app uses Bloc for state management with a repository pattern separating API calls from UI logic.

The admin panel is built with Filament, giving the operations team a full-featured back office: inventory updates, order processing, customer lookups, promo code generation, and sales reporting — without any custom code for each new feature.

What shipped

A production retail platform on iOS and Android: full product catalog, variant-aware cart, multi-payment checkout, real-time order tracking, push notification alerts, and a complete admin panel. All maintained and updated by a single developer — me.

The outcome

Lootah Shop operates as a standalone retail channel with no external dependency on any third-party platform. The client owns the code, the data, and the infrastructure. Every feature added since launch has been shipped in days, not weeks, because the architecture was built right from the start.
